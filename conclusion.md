# Project Conclusion

This project allowed us to investigate how [Semantic Web](https://it.wikipedia.org/wiki/Web_semantico) technologies and [Large Language Models (LLMs)](https://en.wikipedia.org/wiki/Large_language_model) can cooperate to document cultural heritage. By studying the [**Basilica di San Petronio**](https://it.wikipedia.org/wiki/Basilica_di_San_Petronio) in [Bologna](https://it.wikipedia.org/wiki/Bologna) and comparing its data structure with the [**Basilica di San Francesco**](https://it.wikipedia.org/wiki/Basilica_di_San_Francesco_(Arezzo)) in [Arezzo](https://it.wikipedia.org/wiki/Arezzo) within the [ArCo Knowledge Graph](https://dati.beniculturali.it/arco/index.php), we identified several data gaps.

During our work, we faced and resolved three main challenges related to data creation, [AI](https://it.wikipedia.org/wiki/A.I._-_Intelligenza_artificiale) validation, and web development.

## ⚠️ Challenges & Solutions

**1️⃣ Prompting [LLMs](https://en.wikipedia.org/wiki/Large_language_model) for Complex [RDF Triples](https://en.wikipedia.org/wiki/Semantic_triple) (Contacts & Tickets)**

•	**Challenge:** When we asked the [AI](https://it.wikipedia.org/wiki/A.I._-_Intelligenza_artificiale) to generate [RDF triples](https://en.wikipedia.org/wiki/Semantic_triple) for information with multiple values, specifically for the ticket types and contact details of the Basilica, the [LLM](https://en.wikipedia.org/wiki/Large_language_model) struggled to structure the code correctly. It was difficult to find the right prompt to get a single subject connected to multiple objects using [ArCo](https://dati.beniculturali.it/arco/index.php) properties without creating errors.

•	**Solution:** We solved this by rewriting our instructions to the [AI](https://it.wikipedia.org/wiki/A.I._-_Intelligenza_artificiale) very clearly. We explicitly told the model how to handle multiple objects for a single predicate, using concrete examples. This helped the [LLM](https://en.wikipedia.org/wiki/Large_language_model) generate correct [Turtle](https://en.wikipedia.org/wiki/Turtle_(syntax)) code for both ticket prices and contact information.

**2️⃣ Evaluating AI-Generated [SPARQL](https://it.wikipedia.org/wiki/SPARQL) Queries for [ArCo](https://dati.beniculturali.it/arco/index.php)**
   
•	**Challenge:** We needed to verify if the [SPARQL](https://it.wikipedia.org/wiki/SPARQL) queries generated by [ChatGPT](https://chatgpt.com) and [Gemini](https://gemini.google.com/app/1c7f4426e431da15) were actually compatible with the [ArCo](https://dati.beniculturali.it/arco/index.php) ontology rules. AI models often invent properties or use generic vocabularies that do not match the specific vocabulary used by [ArCo](https://dati.beniculturali.it/arco/index.php) for Italian cultural heritage.

•	**Solution:** We manually reviewed every query proposed by the [AI](https://it.wikipedia.org/wiki/A.I._-_Intelligenza_artificiale). We cross-checked the properties with the official [ArCo](https://dati.beniculturali.it/arco/index.php) documentation and compared them with the working queries from our benchmark site, the [Basilica of Arezzo](https://it.wikipedia.org/wiki/Basilica_di_San_Francesco_(Arezzo)). This verification step ensured that our queries were accurate and functional.

**3️⃣ Setting up and Linking the Website on [GitHub](https://github.com) Pages**

•	**Challenge:** As absolute beginners in web development, creating the project website was a central obstacle. We found it very difficult at first to understand how to create a new [GitHub](https://github.com) repository from scratch, adapt existing Jekyll templates, and manually create all the navigation hyperlinks to connect the pages together.

•	**Solution:** We studied the basic documentation for [GitHub](https://github.com) and Jekyll step by step. We took an existing template, carefully analyzed its structure, and slowly learned how to modify the code. Through continuous team testing, we successfully created the repository and built a working navigation system between all our pages.

## 📍 Final Reflections

•	**The Importance of Benchmarking:** Comparing our project on the [Basilica di San Petronio](https://it.wikipedia.org/wiki/Basilica_di_San_Petronio) with the [Basilica di San Francesco](https://it.wikipedia.org/wiki/Basilica_di_San_Francesco_(Arezzo)) in [Arezzo](https://it.wikipedia.org/wiki/Arezzo) was a fundamental step for our team. Because the [ArCo](https://dati.beniculturali.it/arco/index.php) ontology is complex and includes many different modules, it is easy to get lost when looking at a monument from scratch. By using this well-documented church as a guide, we could easily see what information was missing for our main topic. This strategy taught us that in semantic web projects, looking at how similar entities are already modeled is the best way to avoid mistakes and follow the correct standards.

•	**Human-in-the-Loop** Our experience with [Gemini](https://gemini.google.com/app/1c7f4426e431da15) and [ChatGPT](https://chatgpt.com) showed us both the power and the limitations of [Large Language Models](https://en.wikipedia.org/wiki/Large_language_model). [AI](https://it.wikipedia.org/wiki/A.I._-_Intelligenza_artificiale) was incredibly helpful for writing code quickly, but it cannot work completely alone in cultural heritage projects. When the models gave us conflicting images for the Basilica or made mistakes with complex data like tickets and contacts, we realized that human judgment is irreplaceable. We had to act as supervisors, checking every detail against official documents to make sure the final data was true, logical, and accurate.

•	**Semantic Contribution:** Ultimately, the most rewarding part of this project was filling real data gaps for the [Basilica di San Petronio](https://it.wikipedia.org/wiki/Basilica_di_San_Petronio). By generating precise RDF triples in [Turtle syntax](https://en.wikipedia.org/wiki/Turtle_(syntax)) according to [ArCo](https://dati.beniculturali.it/arco/index.php) standards, we created structured data that is ready to be integrated into the [semantic web](https://it.wikipedia.org/wiki/Web_semantico). This contribution helps make digital cultural heritage more complete and accessible for everyone.
